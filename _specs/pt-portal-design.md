# PTポータル 設計方針・UI/UX決定事項

> ステータス: 実装中（残課題あり → 下記「残課題」参照）  
> 最終更新: 2026-03-06  
> 確認者: Tim

---

## 1. 設計方針の決定（Tim確認済み）

### PTポータルはhub.html一本化（2026-03-06 決定）

**決定:** `coreto-partner.html`（独立ポータル）を廃止し、`coreto-hub.html` 内のPTダッシュボードに一本化する。

**理由:** 同じ機能（案件/紹介コード/報酬/メッセージ/新規登録）がhub.htmlとpartner.htmlの両方に実装されており、PTユーザーが「どちらが正の画面か」混乱する構造だったため。

**Tim がhub.htmlのUIを気に入っている** ので、hub.htmlを正のUI基盤として採用する。

**ステータス:** `partner.html` は現在も残存しているが、PTの主な使用画面はhub.html。将来的に `partner.html` は完全廃止またはHQ管理用途のみに整理する。

## 2. hub.html PTの画面構成（現在の実装）

### サイドバー構成

```
【メイン】
  📊 ダッシュボード         → hub内PTダッシュボード
  🎓 オンボーディング研修   → coreto-onboarding.html（外部リンク）
  📋 紹介案件              → hub内案件一覧
  🔗 紹介コード・リンク     → hub内紹介コードページ
  💰 報酬・振込明細        → hub内報酬ページ
  ➕ 新規紹介を登録         → hub内登録フォーム

【アカウント】
  💬 LINE連携              → coreto-line-link.html（外部リンク）
  🪪 eKYC・本人確認         → coreto-kyc.html（外部リンク）
```

### ダッシュボード KPI（4枚）

- 紹介案件（累計）
- 進行中
- 成約件数
- 累計報酬

## 3. 実施済み変更（コミット 567a076）

### hub.html — 業務ツールカード削除

`reDashPage`（REエージェントダッシュボード）最下部にあった業務ツールクイックランチ（IT重説・内見申込・成約クローズ・領収書の4ボタン）を削除。

**理由:** 担当案件テーブルの各行に「対応 ↗」ボタンが既にあり、機能が重複していた。Tim の判断により不要と確定。

### hub.html — PTサイドバー整理

「紹介」セクション（紹介リンク発行）を削除。メインの「紹介コード・リンク」と完全重複のため整理。

### onboarding.html — PTタスクリンク先をhubに統一

`TASKS_PT` 内の `coreto-partner-feed.html` リンク4箇所を `coreto-hub.html` に変更。hub一本化に合わせた整合性修正。
