# 案件マッチング仕様

最終更新: 2026-03-02

---

## 1. マッチング方法（パートナー選択）

パートナーが案件を投稿する際、2つのマッチング方法から選択する。

| 方法 | 説明 |
|---|---|
| **ボード掲載（先着）** | 案件ボードに掲載 → 先着のAGが「対応する」を押すと即マッチング成立 |
| **指名** | 知り合いのAGのIDを指定 → AGが承認するとマッチング成立 |

## 2. ボード掲載フロー

```
① パートナーが「ボード掲載」を選択して案件投稿
      ↓
② 案件ボードに掲載（顧客情報は非開示）
      ↓
③ AGが「対応する」を押す（先着1名）→ 即マッチング成立
      ↓ 顧客情報が担当AGに開示
④ HQがLINE WORKSトークルームを作成
      ↓
⑤ 担当AGが顧客対応開始
```

## 3. 指名フロー

```
① パートナーが「指名」を選択 → AGのIDと指名メッセージを入力して投稿
      ↓
② 指名されたAGのhubに通知 → AG が承認 or 辞退
      ↓ 承認
③ マッチング成立 → 顧客情報が担当AGに開示
      ↓
④ HQがLINE WORKSトークルームを作成
      ↓
⑤ 担当AGが顧客対応開始
```

## 4. LINE WORKSトークルーム作成

### HQの作業手順

マッチング確定時、Slackに以下が自動通知される。

```
🔔 案件マッチング確定
AG（LINE検索用）: `山田 誠 AG-10023`
PT（LINE検索用）: `佐藤 美咲 PT-30007`
ルーム名: `【案件】池袋1K_0226_AG-10023`
最初のメッセージ: [コピー用テンプレート]
```

HQの操作：
1. LINE WORKSを開く
2. 新規トーク → `山田 誠 AG-10023` で検索して選択
3. `佐藤 美咲 PT-30007` を追加
4. ルーム名をコピペ
5. 最初のメッセージをコピペして送信

**所要時間：約1分**

### LINE WORKS連絡先管理ルール
- AGとパートナーは全員 **本名 + CORETO ID** でLINE連絡先に登録する
- 例: `山田 誠 AG-10023`、`佐藤 美咲 PT-30007`
- 入会時の友達追加時点で登録を完了させる

### 自動化の制約
LINE WORKS Bot APIはLINEユーザー（AG・PT）をAPIで自動追加できないため、
HQによる手動作成（1分程度）で対応する。

## 5. マッチング不成立時のHQ対応

| 状況 | HQのアクション |
|---|---|
| ボード掲載24時間以内に応募なし | Slackで全AG向けに呼びかけ |
| 指名AGが辞退 | パートナーに通知 → ボード掲載に切り替えを提案 |
| それでも不成立 | HQが直接AGを指定してアサイン |

## 6. 顧客情報の開示タイミング

- ボード掲載・指名ともに **マッチング成立後に担当AGへのみ開示**
- パートナーの顧客情報はマッチング確定前はシステム上で保護
