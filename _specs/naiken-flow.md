# 内見依頼フロー仕様
最終更新: 2026-03-03

## 1. 内見依頼書の発行
AGがpipeline.htmlの案件カードから発行する。

フロー:
1. pipeline → 内見依頼書を発行
2. naiken.html で物件名・管理会社・希望日時（第1〜3候補）を入力
3. Slack #内見完了 に通知
4. 管理会社に連絡（システム外: 電話/メール）
5. 内見日時確定 → システムに確定日を入力 → naikenDone()
6. 次ステップへ（賃貸: 申込 / 売買: 買付 / 三為: 業者確認）

## 2. 内見スキップ（売買・三為のみ）
クライアントが現地確認済みの場合、内見をスキップして直接買付へ進める。

## 3. 注意事項
- 管理会社への連絡はシステム外で行う
- 内見完了日はシステムに必ず入力（ランクポイント計算の基準日）
