## 前提・役割定義

| 役割 | できること | できないこと |
|---|---|---|
| エージェント | 全工程（宅建業務含む）※内見・重説・契約等 | — |
| パートナー | クライアント紹介・進捗確認・連絡のみ | 内見・IT重説・契約等の宅建業務 |
| HQ | 案件管理・アサイン・成約承認・報酬計算 | — |

**重要：賃貸仲介においてパートナーが案件を自己完結することはない。**  
理由：内見等の宅建業務はパートナーが行えないため、必ずエージェントと組む。

**管理会社の入居審査はCORETOのフローに含まれない。**  
申込後に管理会社側で実施されるものであり、HQ・エージェントは関与しない。

## 報酬分配

| パターン | エージェント | パートナー | HQ |
|---|---|---|---|
| 自己持ち込み | 65〜75%（ランクによる） | — | 25〜35% |
| マッチング（パートナー紹介） | **25%** | **25%** | **50%** |

### ランク別報酬率（自己持ち込み時）

| ランク | エージェント | HQ |
|---|---|---|
| Platinum | 75% | 25% |
| Gold | 70% | 30% |
| Silver | 67.5% | 32.5% |
| Bronze | 65% | 35% |

### 源泉徴収
- エージェント・パートナーともに報酬から源泉徴収（10.21%）を控除して振込
- 振込日：翌月25日

## フロー詳細

### ① 自己持ち込み案件（エージェント視点）

```
1. クライアント獲得（エージェント自身が開拓）

2. 案件登録
   └─ coreto-pipeline.html で新規案件作成
       → Kintone にレコード作成
       → Slack #新規案件 に通知

3. 内見手配・立会い
   └─ coreto-naiken.html で予約・スケジュール管理
       → クライアントへ日程通知

4. 申込書類収集
   └─ coreto-batch-close.html で書類管理
   ※ 申込後の入居審査は管理会社が実施（CORETOは関与しない）

5. IT重説実施
   └─ coreto-it-setsu.html でZoom/対面設定・記録
       → coreto-itsetsu-schedule.html で日程調整
       → クライアント・貸主に通知

6. 契約締結
   └─ 契約書確認・署名

7. 成約報告
   └─ coreto-batch-close.html から報告送信
       → HQが承認（coreto-case-mgmt.html）
       → Kintone 成約ステータス更新

8. 報酬振込（翌月25日）
   └─ coreto-hub.html → 「今月の報酬・振込明細」で確認
```

**使用ページ一覧**

| ステップ | ページ | 担当 |
|---|---|---|
| 案件登録 | coreto-pipeline.html | エージェント |
| 内見管理 | coreto-naiken.html | エージェント |
| 書類収集・成約報告 | coreto-batch-close.html | エージェント |
| IT重説 | coreto-it-setsu.html | エージェント |
| 重説日程調整 | coreto-itsetsu-schedule.html | エージェント |
| 成約承認・案件監視 | coreto-case-mgmt.html | HQ |
| 報酬確認 | coreto-hub.html（paymentページ） | エージェント |

### ② マッチング案件（パートナー紹介）

#### パートナー視点

```
1. 知人・顧客から「物件を探している」相談を受ける

2. CORETOへ紹介登録
   └─ coreto-hub.html → 「新規紹介を登録」
       → Slack #紹介案件 に通知（HQへ）
       → Kintone にレコード作成

3. 担当エージェントアサイン（HQが実施）
   └─ アサイン完了をパートナーに通知

4. 進捗確認
   └─ coreto-hub.html → 「紹介案件」タブで確認
       → 担当エージェントとメッセージでやり取り可能

5. 成約後、翌月25日に報酬振込
   └─ coreto-hub.html → 「報酬・振込明細」で確認
```

#### HQ視点

```
1. Slack #紹介案件 で紹介通知を受け取る

2. エージェントをアサイン
   └─ coreto-matching.html
       → エージェントのランク・稼働状況を確認して選定
       → エージェントに通知

3. 案件進捗の監視・介入
   └─ coreto-case-mgmt.html
       → 遅延・問題があれば介入・サポート

4. IT重説の記録管理
   └─ coreto-itsetsu-hq.html

5. 成約報告の承認
   └─ coreto-case-mgmt.html

6. 月次締め・報酬確定
   └─ coreto-payroll.html
       → エージェント報酬（25%）確定
       → パートナー報酬（25%）確定
       → 翌月25日に一括振込
```

#### エージェント視点

```
1. HQからアサイン通知を受け取る

2. クライアントへ連絡・ヒアリング

3. 以降は自己持ち込みと同じ宅建業務フロー
   └─ 内見手配 → 申込書類収集 → IT重説 → 契約締結

4. 成約報告
   └─ coreto-batch-close.html から報告送信

5. 翌月25日 振込（手数料の25%）
```

**使用ページ一覧（マッチング追加分）**

| ステップ | ページ | 担当 |
|---|---|---|
| 紹介登録 | coreto-hub.html（pt-register） | パートナー |
| 進捗確認 | coreto-hub.html（pt-cases） | パートナー |
| 報酬確認 | coreto-hub.html（pt-rewards） | パートナー |
| エージェントアサイン | coreto-matching.html | HQ |
| 案件監視・介入 | coreto-case-mgmt.html | HQ |
| IT重説管理 | coreto-itsetsu-hq.html | HQ |
| 月次報酬計算 | coreto-payroll.html | HQ |

## ③ HQ起票案件（HQ自己持ち込み）

HQが直接クライアントを獲得した場合。パートナーは関与しない。

```
1. HQがクライアントを直接獲得

2. pipeline.html でHQ起票モードにて案件登録
       → エージェントを指名アサイン

3. アサインされたエージェントが宅建業務を全て担当
   └─ フローは①（自己持ち込み）と同一

4. 成約報告 → HQが承認 → 報酬計算
   └─ パートナー報酬は発生しない
   └─ エージェント報酬はランク報酬率に従う
```
