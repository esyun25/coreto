# 売買・三為成約報告フロー仕様

最終更新: 2026-03-03

---

## 1. 対象案件種別

| 種別 | 説明 | パイプラインID |
|---|---|---|
| **売買仲介** | 一般の売買取引。買主または売主側の仲介 | type: `reSale` |
| **三為（転売）** | 自社が中間取得して転売。業者確認ステップあり | type: `sannai` |

---

## 2. 売買仲介フロー

```
内見
  ↓ naikenDone()
買付受領（offer）— 物件名・売買価格・買付日を入力
  ↓ offerDone()
請求書発行（invoice）— 種別選択（commission / AD+ / ADonly）
  ↓ invoiceDone()
契約締結（contract）— 売買契約日・契約書スキャン
  ↓ saleContractDone()
決済・引渡し（settlement）— 決済日入力
  ↓ settlementDone() → 成約完了（done）
```

### 各ステップの入力内容

| ステップ | 入力項目 | 備考 |
|---|---|---|
| 買付 | 物件名・売買価格・買付日 | 売買価格は報酬上限計算に使用 |
| 請求書 | 種別・仲介手数料（複数行可）・AD料 | 売買価格×3%+6万円+消費税が法定上限 |
| 契約 | 売買契約日・契約書PDF | 契約書はGoogle Driveにアップロード |
| 決済 | 決済日 | 成約報告の基準日（賃発日相当） |

### 手数料の法定上限チェック（宅建業法46条）

```
売買価格 ≤ 200万円  → 上限 = 売買価格 × 5% + 消費税
売買価格 ≤ 400万円  → 上限 = 売買価格 × 4% + 2万円 + 消費税
売買価格 > 400万円  → 上限 = 売買価格 × 3% + 6万円 + 消費税（通常ケース）
```

手数料が上限を超えると請求書発行フォームでエラー表示。

## 3. 三為（転売）フロー

```
内見（任意スキップ可）
  ↓
業者確認（verify）— 相手業者の宅建業者番号を確認・本部へSlack通知
  ↓ 本部が認証コード発行
買付（offer）— 仕入価格・販売価格・仲介手数料（売り側/買い側）・業務委託手数料
  ↓ offerDone()
請求書（invoice）— 買主向け必須・売主向け任意
  ↓
契約（contract）— 商流図確認・売主/買主/仲介業者/紹介者を登録
  ↓ sannaiContractDone()
決済（settlement）— 決済日入力
  ↓ settlementDone() → 成約完了（done）
```

### 三為固有の追加入力

| 項目 | 内容 |
|---|---|
| 仕入価格 | 売主から購入する価格 |
| 販売価格 | 買主に売却する価格 |
| 粗利 | 販売価格 − 仕入価格 − 仲介手数料の自動計算表示 |
| 売り側仲介業者 | 複数登録可 |
| 買い側仲介業者 | 複数登録可 |
| 紹介者 | 所属側（売り/買い）・種別（宅建業者/一般法人/個人） |

## 4. 成約後の報酬計算トリガー

`settlementDone()` 実行時に以下が起動する。

```
① case.step = 'done' に更新
② Slack #決済完了 に通知（物件名・決済日・販売価格・請求合計）
③ batch-close.html（成約報告フォーム）へ遷移促進
```

成約報告フォーム（coreto-batch-close.html）では追加で：

```
④ 業種（売買 or 三為）・決済日・手数料・三為フラグ（三為補正）を入力
⑤ ランク報酬率を適用して報酬額を計算
⑥ 月次振込スケジュールに登録（25日払い）
⑦ HQ payroll.html の当月振込リストに反映
```

### 三為補正とは

三為案件は価格が高額になるため、成約報告フォームで「三為フラグ」を立てると報酬計算に専用補正率が適用される（commission-rank-mentor.md参照）。

---

## 5. 手数料引下げ時のルール

売買手数料を法定上限より低く設定する場合、理由入力が必須（incoming-payments.md参照）。

理由例:
- 複数回成約クライアントへのサービス
- 物件価格の特殊事情
- 紹介元への便宜

## 6. Slack通知チャンネル構成

| タイミング | チャンネル | 主な内容 |
|---|---|---|
| 内見完了 | #内見完了 | 物件名・クライアント・担当AG |
| 三為業者確認 | #三為業者確認 | 業者名・宅建業者番号・認証コード発行依頼 |
| 請求書発行 | #請求書発行 | 手数料金額・GMO振込先口座 |
| 契約締結 | #契約締結 | 売買契約日・商流（三為の場合） |
| 決済完了 | #決済完了 | 決済日・販売価格・請求合計 |

## 7. AI活用ポイント（予定）

### ② 成約報告Slack通知文の自動生成
- **発火**: `settlementDone()` 実行時
- **処理**: 案件データ（物件名・価格・担当AG・日付）をHaikuに渡し、Slack投稿用の整形文を生成
- **目的**: フォーマットの統一・抜け漏れ防止
- **コスト目安**: 約200トークン/件 → 月50件で月数十円

---

## 8. キャンセル処理

`cancelCase()` でいつでもキャンセル可能。`reactivate()` で再開可能。キャンセル理由の入力は現状任意。
