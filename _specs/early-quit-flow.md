# 早期退職アラート・返金フロー仕様

最終更新: 2026-03-03  
対象業種: **HR（人材紹介）専用**

---

## 1. 概要

HR案件は成約後に2段階での報酬支払いがある。入社後一定期間内に退職した場合、企業との合意内容に基づき報酬を返金する義務が発生する。本フローはその管理・通知・返金処理を規定する。

---

## 2. 返金保証の登録（AGの義務）

### タイミング
成約時（入社確定後）に担当AGが企業との合意内容を登録する。

### 保証タイプ
| タイプ | 内容 |
|---|---|
| **保証あり** | 在職〇ヶ月以内に退職した場合に返金率を段階設定 |
| **保証なし** | 入社確定で全額支払い。返金義務なし |

### 保証条件の例
```
3ヶ月以内退職 → 100%返金
6ヶ月以内退職 → 50%返金
6ヶ月超       → 返金なし
```

企業・求人ごとに条件が異なるため、**案件ごとに個別登録**が必須。

### 登録箇所
`coreto-early-quit.html` → 「返金条件登録」モーダル  
または `coreto-pipeline.html` → HR案件の成約報告時

---

## 3. 在籍確認フロー（第2回報酬発生条件）

```
入社日
  ↓ 在籍確認期限（入社から〇ヶ月後）
担当AGが在籍確認 → HQが確認
  ↓ 在籍確認OK
第2回報酬を月次振込に組み込み（翌25日払い）
  ↓ 在籍確認NG or 期限超過
第2回報酬を翌月に延期 → 再確認
```

### 在籍確認アラート（Zapier自動チェック）
- 在籍確認期限7日前: AGとHQにSlack通知
- 在籍確認期限3日前: 緊急アラート（HQダッシュボードにバナー表示）
- 在籍確認期限超過: 警告表示・第2回支払い延期

---

## 4. 早期退職発生フロー

```
① 担当AGが退職を把握
      ↓
② coreto-early-quit.html で退職報告
   入力: 退職日・退職理由（任意）
      ↓
③ Slack #早期退職報告 に自動通知
   （候補者名・担当AG・企業名・退職日・返金計算結果）
      ↓
④ HQが返金額を確認・企業に連絡
      ↓
⑤ 企業から返金入金を受領（期限: 退職確認後30日以内が目安）
      ↓
⑥ 担当AGの報酬から差し引き（月次振込で控除）
      ↓
⑦ 返金完了・案件クローズ
```

---

## 5. 返金額計算ロジック

```
在職期間 = 退職日 - 入社日（日数 / 30.44 = ヶ月換算）
該当保証段階 = 在職期間 < 〇ヶ月 の条件に最初にヒットした段階
返金額 = 総純利益 × 返金率(%)
```

### 返金の報酬控除タイミング
- 返金発生月の翌25日払いから控除
- 控除額が当月報酬を超える場合は翌月以降に繰り越し

---

## 6. HQの対応フロー（ステータス管理）

| ステータス | 意味 | 次のアクション |
|---|---|---|
| pending | 退職報告あり・返金交渉中 | 企業に連絡・返金額確定 |
| claimed | 返金請求済み | 入金待ち |
| received | 返金入金確認済み | AG報酬から控除処理 |
| closed | 処理完了 | 案件アーカイブ |
| disputed | 企業が返金を拒否 | HQ判断・法的対応検討 |

## 7. AI活用ポイント（予定）

### ④ 早期退職返金文面生成
- **発火**: HQが「返金請求済み」ステータスに変更したとき
- **処理**: 案件情報（候補者名・企業名・入社日・退職日・返金額）をHaikuに渡し、企業向け返金請求レターの下書きを生成
- **目的**: 担当者による文面のブレ防止・作成時間削減
- **コスト目安**: 約500トークン/件 → 月10件で月数十円
