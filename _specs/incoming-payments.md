# 手数料・報酬入金 仕様書

> ステータス: 確定  
> 最終更新: 2026-02-28  
> 確認者: Tim

---

## 1. 入金の種類と請求先

CORETOが受け取る入金は案件種別によって請求先・タイミング・名目が異なる。

| 案件種別 | 請求項目 | 請求先 | 入金タイミング |
|---|---|---|---|
| **賃貸仲介** | 仲介手数料 | 借主（顧客） | 重説・鍵渡し前 |
| **賃貸仲介** | 広告料（AD料） | 管理会社 | 成約後 随時 |
| **売買仲介** | 仲介手数料 | 買主 or 売主 | 決済時 |
| **三為取引** | 仲介手数料・業務委託料 | 買主・管理会社等 | 決済時 |
| **人材紹介** | 紹介手数料 | 紹介先企業 | 入社後 随時 |

---

## 2. 請求書の発行

### 発行者
HQが発行する（coreto-hq-receipt.html）

### 発行タイミング（案件ステップ）

**賃貸仲介**
```
内見 → 申込 → 審査 → [請求書発行] → 重説・入金 → 鍵渡し → 完了
```

**売買仲介 / 三為取引**
```
内見 → 買付 → [請求書発行] → 契約 → 決済 → 完了
```

**人材紹介**
```
エントリー → 選考 → 内定 → 入社 → [請求書発行] → 入金 → 完了
```

### 請求書の項目（複数設定可・最大5項目）

| 項目名 | 主な請求先 |
|---|---|
| 仲介手数料 | 顧客（借主・買主） |
| 広告料（AD料） | 管理会社 |
| 業務委託料 | 管理会社・法人 |
| 紹介手数料 | 紹介先企業 |
| その他 | 任意 |

## 3. 案件種別ごとの入金フロー

### 賃貸仲介

**仲介手数料（借主から）**
```
審査通過・入居確定
  ↓
HQが請求書発行（仲介手数料）
  ↓
借主が重説前までに振込（入金期限あり）
  ↓
GMO BaaS API webhook → Kintone 入金フラグ自動更新
  ↓
入金確認後、IT重説・鍵渡しへ進む
```

**広告料（管理会社から）**
```
成約確定
  ↓
HQが管理会社に請求書発行（広告料）
  ↓
管理会社が振込（入金タイミングは管理会社次第）
  ↓
入金確認後、Kintoneに記録
```

**重要：入金前に鍵渡しをしてはいけない**
仲介手数料の全額入金確認が取れていない状態での鍵渡しは禁止。
エージェントは coreto-batch-close.html に全額入金確認日を入力後に成約報告を送信できる。

---

### 売買仲介

**仲介手数料（買主・売主から）**
```
売買契約締結
  ↓
HQが請求書発行
  ↓
決済日に入金（司法書士立会い・所有権移転と同日）
  ↓
GMO BaaS API webhook → Kintone 入金フラグ自動更新
  ↓
成約処理
```

**手数料上限（宅建業法）**
```
売買価格 × 3% + 6万円（+ 消費税）× 2（売主・買主双方から受領する場合）
例：3,000万円の物件 → 上限 (3,000万 × 3% + 6万) × 1.1 = 105.6万円
```

---

### 三為取引（転売）

売買仲介に準じるが、CORETOが中間で物件を取得・転売するため手数料率に15%の補正が入る（エージェント報酬率から15%減算）。

---

### 人材紹介

**紹介手数料（紹介先企業から）**
```
候補者の入社確定
  ↓
HQが企業に請求書発行
  ↓
企業が入金（月末締め翌月払い等、企業の支払いサイクルによる）
  ↓
GMO BaaS API webhook → Kintone 入金フラグ自動更新
  ↓
エージェントへの報酬は2回分割（入社翌月25日・保証期間経過後翌月25日）
```

**紹介手数料の相場**
理論年収（想定年収）の30〜35%程度が業界標準。CORETOの設定はTim確認。

## 4. 入金手段と管理

### 入金手段
**GMOあおぞらネット銀行 バーチャル口座への銀行振込**

クライアント・企業・管理会社はすべて銀行振込で対応。カード決済・PayPay等は不使用。

### 入金自動検知（本番実装後）

```
クライアントが振込
  ↓
GMO BaaS API webhook 発火
  ↓
Zapier が受信
  ↓
Kintone の該当レコードに入金フラグ・入金日を自動書き込み
  ↓
Slack #入金確認 チャンネルに通知
  ↓
エージェントのハブに「入金確認 → 成約報告を送信してください」通知
```

### 入金期限の設定

成約報告送信時（coreto-batch-close.html）に入金期限を設定できる。期限超過案件はSlackで自動アラート。

## 6. 入金確認〜報酬支払いの全体フロー

```
クライアントが仲介手数料を振込
          ↓
GMO webhook → Kintone 入金フラグON → Slackに通知
          ↓
エージェントが全額入金確認日を入力して成約報告
（coreto-batch-close.html）
          ↓
Kintoneにレコード登録・Slack #成約報告 に通知
          ↓
月末締め（8項目自動チェック）
          ↓
HQ最終承認（翌月〜10日）
          ↓
支払明細書 自動発行（翌月15日）
          ↓
コミッション振込（翌月25日 or 前営業日）
  ├─ 〜99万円 → Kyash
  └─ 100万円〜 → GMO振込CSV
```

## 7. 確定事項

### 人材紹介の手数料率
- 業界標準は理論年収の**30%程度**
- 企業によって異なるため案件ごとに設定（固定値なし）
- coreto-batch-close.html で入力時に手動設定できるようにする

### 売買仲介の手数料
- デフォルト：宅建業法上限（売買価格 × 3% + 6万円）× 1.1（消費税）
- **引き下げ可能**：下げる場合は理由記載を必須とする
- 理由例：「競合他社との競争のため」「長期顧客のため割引」等
- ※三為取引（中間転売）は売買案件の一種。担当エージェントの報酬率から15%減算する補正が別途発生する

### AD料（広告料）の入力
- **実際の金額を直接入力**（0.1〜5ヶ月分と幅広いため固定計算式なし）
- coreto-batch-close.html の請求項目「広告料（AD）」欄に金額を手入力
- 相場目安：賃料の0.1〜5ヶ月分（管理会社・物件・市況による）

### 入金期限超過アラートの通知先
- **HQ**（Slack #入金督促）
- **担当エージェント**（Slack DM or ハブ通知）
- 両方に同時通知する
