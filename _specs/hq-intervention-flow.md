# HQ案件介入・強制変更フロー仕様
最終更新: 2026-03-03

## 1. HQが介入すべきケース
- 案件が7日以上同一ステップで停滞（高優先）
- AGから困っているとの連絡（高優先）
- マッチング不成立で手動アサインが必要（高優先）
- クライアントからのクレーム（高優先）
- 担当AGの急な離脱・辞退（高優先）
- HQ起票案件の進捗確認（中優先）
- 月末締め前のステップ確認（中優先）

## 2. 介入操作の種類（coreto-case-mgmt.htmlから実行）
- ステップ強制変更: 任意のステップに変更
- 担当AG変更: 別AGをアサイン
- HQメモ追加: 介入理由・対応内容を記録
- フラグ設定: 要注意・保留・VIP等
- キャンセル / 案件復活
- CSV出力: 全案件エクスポート
- HQ起票: HQが直接新規案件を作成

## 3. 介入後の記録
全操作が活動履歴に自動記録される（操作日時・操作者・内容・理由）。

## 4. 担当AG変更時の通知
1. 旧担当AGにSlack DMで通知
2. 新担当AGにSlack DMで通知（案件概要・引継ぎ内容）
3. パートナー案件の場合はPTにもLINEで通知

## 5. 関連ファイル
- coreto-case-mgmt.html: HQ案件介入管理
- coreto-pipeline.html: 案件管理（AG向け）
- coreto-matching.html: マッチング管理
