> ステータス: 確定  
> 最終更新: 2026-02-28  
> 確認者: Tim

## 全体カレンダー（毎月）

```
【当月中】
  随時    クライアントから仲介手数料入金（GMOバーチャル口座）
  随時    エージェントが成約報告（全額入金確認後のみ）
  随時    即時払い申請（2FA必須）→ HQ処理30分〜2時間以内
  毎月    Stripeで月会費引き落とし（Bronze¥980 / Silver¥490）

【月末】
  月末日  月次締め処理（8項目自動チェック）

【翌月】
  1日     ランク自動判定・Stripeプラン変更
  〜10日  HQ最終承認（coreto-batch-close.html）
  15日    支払明細書 自動発行（エージェント・パートナーに送付）
  20日    GMO振込CSVデータ作成・確認
  25日    コミッション一括振込（Kyash or GMO銀行）
```

## 1. クライアントからの仲介手数料入金

### 入金手段
銀行振込（GMOあおぞらネット銀行バーチャル口座）

### タイミング
成約・契約締結後、請求書をクライアントに発行してから**随時**。入金タイミングはクライアント次第のため、都度確認。

### 入金確認フロー

```
クライアントが振込
  ↓
GMO BaaS API webhook → Zapier → Kintoneの入金フラグを自動更新
  ↓
Slackの #入金確認 チャンネルに通知
  ↓
エージェントが coreto-batch-close.html で全額入金確認日を入力して成約報告
```

### 重要ルール
- **全額入金確認後でないと成約報告を送信できない**
- 入金確認日が翌月25日の振込基準日になる
- 分割払い・手付金先払いのケースでも最終入金確認日を基準とする

## 2. エージェントへのコミッション支払い（月次通常振込）

### 振込日：**毎月25日**

### 月次処理スケジュール

| 日程 | 処理内容 | 担当 | ページ |
|---|---|---|---|
| 月末日 | 月次締め・8項目自動チェック実行 | 自動（Kintone） | coreto-batch-close.html |
| 翌月1〜10日 | HQ最終承認・エラー対応 | HQ | coreto-batch-close.html |
| 翌月15日 | 支払明細書 自動発行・送付 | 自動 | coreto-payment.html |
| 翌月20日 | GMO振込CSVデータ作成・最終確認 | HQ | coreto-batch-close.html |
| 翌月25日 | 振込実行 | 自動（Kyash）/ 手動（GMO） | — |

### 月末8項目自動チェック内容

| # | チェック項目 |
|---|---|
| ① | 全額入金確認（全件入金済みか） |
| ② | 未報告案件の有無（※要確認：実装上②は何か） |
| ③ | 報酬額の異常値検出 |
| ④ | 源泉徴収計算の正確性 |
| ⑤ | 重複成約チェック |
| ⑥ | 成約日の妥当性 |
| ⑦ | （※要確認：実装上⑦は何か） |
| ⑧ | 振込先口座確認（登録済み・有効か） |

### 振込方法の分岐

```
振込額 < 100万円 → Kyash法人送金（自動・口座情報不要）
振込額 ≥ 100万円 → GMO振込CSV（手動・HQが実行）
```

### 賃貸仲介の計算式（振込額）

**自己持ち込み案件**
```
振込額 = 手数料 × ランク報酬率 − 源泉徴収（10.21%）− システム利用料
```

**パートナー案件**
```
エージェント振込額 = 手数料 × 25% − 源泉徴収 − システム利用料の半額
パートナー振込額  = 手数料 × 25% − 源泉徴収 − システム利用料の半額
メンター振込額    = 手数料 × メンターフィー率 − 源泉徴収（存在する場合）
```

## 3. 人材紹介コミッションの分割払いスケジュール

人材紹介は成約後の入社・定着を確認してから報酬が発生するため、**2回分割払い**となる。

### 分割比率
- 第1回：報酬の**50%**
- 第2回：報酬の**残り50%**

### 各回の振込条件

| 回 | 振込条件 | 振込タイミング |
|---|---|---|
| 第1回（50%） | 全額入金確認済み **かつ** 入社から1週間経過 | 条件成立後の翌月25日 |
| 第2回（50%） | 入社から返戻保証期間経過（3ヶ月 or 6ヶ月） | 保証期間経過後の翌月25日 |

### 返戻保証期間
- 3ヶ月 or 6ヶ月（案件ごとに設定）
- 保証期間内に退職した場合は第2回の支払いなし（一部返金が発生するケースあり）

### 例（報酬¥693,000・保証3ヶ月・1月入社の場合）

```
第1回（¥346,500）→ 2月25日振込（入社1/15・入金確認1/20 → 解禁1/22）
第2回（¥346,500）→ 5月25日振込（入社3ヶ月後4/15 → 解禁4/15）
```

## 4. 即時払い（Instant Pay）

### 概要
翌月25日の通常振込を待たずに、確定済み報酬を即日受け取れる機能。

### 手数料
申請額の **15%**（エージェント負担）

### 申請条件
- 最低申請額：**¥10,000**
- 上限：その時点での確定済み未払い報酬残高
- 2FA（SMS認証）が必須
- account-settings.html で電話番号の登録が必要（未登録は申請不可）

### 人材紹介の即時払い解禁条件

| 回 | 即時払い解禁タイミング |
|---|---|
| 第1回 | 全額入金確認済み **かつ** 入社から1週間後 |
| 第2回 | 入社から1ヶ月後 |

※ 通常振込（25日）より早く解禁されるが、保証期間中は第2回の即時払いは入社1ヶ月後まで不可。

### 処理フロー

```
エージェントが申請（coreto-instant-pay.html）
  ├─ 金額入力（最低¥10,000）
  └─ SMS認証コード入力（10分以内）
      ↓
Slack #即時払い に自動通知
      ↓
HQが管理画面で確認・振込実行
  ├─ 振込額 < 100万円 → Kyashで即時送金
  └─ 振込額 ≥ 100万円 → GMO銀行から手動振込
      ↓
通常30分〜2時間以内に振込完了
      ↓
Kintoneに記録（月次計算時に重複控除）
```

### 月次計算との関係
即時払いで先払いした分は翌月25日の通常振込から自動的に控除される。

## 5. 月会費（Stripe）

### 引き落とし方式
Stripeによる**毎月直接カード引き落とし**（報酬からの天引きは行わない）

### 金額・対象

| ランク | 月会費 | 引き落とし |
|---|---|---|
| Bronze | ¥980/月 | Stripeが毎月引き落とし |
| Silver | ¥490/月 | Stripeが毎月引き落とし |
| Gold | ¥0 | 課金なし |
| Platinum | ¥0 | 課金なし |
| 特別紹介コード適用者 | ¥0 | 永久停止 |

### ランク変動時の切り替えタイミング
毎月1日のランク判定後、Zapier経由でStripeのプランを自動変更。

### カード決済失敗時
Stripeが3回自動リトライ（1日・3日・7日後）後、Slack #請求エラー に通知。

## 6. 支払明細書

### 発行タイミング：毎月**15日**

### 内容
- 対象月の全成約案件一覧
- 各案件の手数料・報酬率・報酬額
- システム利用料
- 源泉徴収額
- 即時払い控除額（利用した場合）
- 最終振込額
- 振込予定日（25日）
- 振込先口座

### 閲覧
- エージェント：coreto-hub.html → 「今月の報酬・振込明細」
- HQ：coreto-payment.html（発行・印刷・PDF保存）
