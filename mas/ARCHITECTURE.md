```
             │ 並列実行
    ┌────────┴─────────────────────────────────┐
    │           Worker Pool (最大5並列)         │
    │  ┌──────────┐  ┌──────────┐  ┌────────┐ │
    │  │ Worker 1 │  │ Worker 2 │  │  ...   │ │
    │  │ Scenario │  │ Scenario │  │        │ │
    │  │   #001   │  │   #002   │  │        │ │
    │  └──────────┘  └──────────┘  └────────┘ │
    └──────────────────────────────────────────┘
             │

    │                                            │
    │  ┌──────────────────────────────────────┐ │
    │  │      Issue Classifier                 │ │

    └────────────────────────────────────────────┘
```

## シナリオ種別（1000パターン）

### 簡単（500件）
- 賃貸仲介: 標準フロー（クライアント→AG→成約→HQ承認→支払）
- 光通信: シンプルな契約フロー
- PT紹介: 基本紹介フロー

### 中難度（300件）
- 人材紹介: フルフロー（面接調整・在籍確認含む）
- IT重説: 宅建士マッチング→重説実施
- 複数AG競合: 同じ案件に複数AGがアプローチ

### 高難度（150件）
- 売買・三為: 複雑な書類フロー
- キャンセル・解約フロー
- PT→AG→HQ の多段承認

### エッジケース（50件）
- 残高不足での即時払い申請
- 期限超過の入金確認
- 重複案件の検出

## 自動修正ポリシー

| 問題種別 | 処理 |
|---|---|
| BUG (コンソールエラー・JS例外) | 自動修正 |
| DATA (ハードコード・空データ) | 自動修正 |
| FLOW (フロー不足・ステップ抜け) | Timに許可申請 |
| UI/DESIGN (レイアウト・デザイン) | Timに許可申請 |
| MISSING (ページ追加・削除) | Timに許可申請 |

## データ活用分析

収集データ:
- 案件種別・成約率・期間
- AG別パフォーマンス
- クライアント属性パターン
- フロー詰まりポイント
- 最適マッチング条件
